JSON Location Tracker
Post URL
This is the full URL you wish for GPS data to POST to. The exact format is discussed in the helpfile. The field resists accidental edits. To edit, tap-and-hold and the field will drop into edit mode.
Continuous Updates
The application can "subscribe" to the GPS system. The GPS readings will get more and more accurate while continously reading, however, it will be pretty hard on the battery and will upload much more data to the webserver.
Update Interval
The GPS can take 3 or 20 seconds to deliver data to the application. This interval is the time between the last read and the start of the next query — as such, the readings may be further apart than this setting might otherwise indicate.
Buffer Size
The application stores readings locally until they're confirmed as accepted by the webserver. This controls the size of the local buffer. If a new reading is "very close" to the last reading, the new timesamp will be added to the last reading, rather than pushing a new reading into the buffer. The list of timestamps in a reading can grow indefinitely, but rarely should if the remote webserver is accepting POSTs.
